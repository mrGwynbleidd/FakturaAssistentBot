How it works

PDF-инструкции
    ↓
извлечение текста
    ↓
разбивка на фрагменты
    ↓
создание embeddings
    ↓
сохранение в Chroma database
    ↓
вопрос пользователя
    ↓
поиск похожих фрагментов
    ↓
Gemini получает context
    ↓
Gemini отвечает
    ↓
вопрос и ответ сохраняются в CSV

Simulating question

Пользователь задал вопрос
    ↓
локальный embedding вопроса
    ↓
локальный поиск в Chroma
    ↓
Gemini API получает context + question
    ↓
Gemini возвращает ответ


What each file do:

app/config.py - read API key from .env

app/rag/pdf_loader.py - read files from data/pdf. Get text from each page and save it as a source

app/rag/splitter.py - split huge text into small one. 10,000 symbouls -> 4 chunks. Increase accuercy

app/rag/embedding.py - embending our text, make vectors

app/rag/index_builder.py - use functions from prev files. Get pdf -> Get its text -> Divid into chunks -> Make embending -> Save it in Chroma

build_index.py - call index builder

app/rag/query_rewriter.py - upgrade in searching, able to answer "I can't login in account" 

!!!app/rag/retriever.py - Find info from Chroma, 	get question -> upgrade it by query_rewriter -> embending question -> find similar chunks in Chroma -> return context and source

app/rag/generator.py - send promt to Gemini, he gets founded info

app/logs/csv_logger.py - save question, answer, time, language, source in app/logs/csv_logger.py

test_pdf_rag.py - test file to check how model handle with different questions, start our pipeline retrieve_context() generate_answer()save_qa()

app/bot/language.py - detect language based on user input

app/core/bot_engine.py - main file will manage whole process, Get question from user, return answer. check question -> detect language -> search context in Chroma -> Give question and context to Gemini -> save q/a in csv -> return result

test_bot_engine.py - file to test how bot_engine works

app/learning/case_logger.py - if model do not found context or answer is weak, it saves quation and model's answer for feature admin review 

app/learning/approved_loader.py - load data from approved.csv, will use in embedding admins question

app/learning/review_manager.py - add admins answer to approved.csv 

test_approve_case.py - file to test how manager review works and check if model learn

Да, для добавления в группу сейчас лучше сделать режим сбора данных, без ответов пользователям.
То есть временно логика будет такая:
Пользователь отправил фото в группу
        ↓
Бот молча сохранил фото
        ↓
Бот записал metadata в CSV
        ↓
Никакого ответа в чат
Важно: чтобы бот видел все сообщения/фото в группе, в BotFather нужно отключить Privacy Mode. Официальная документация Telegram указывает, что с включённым privacy mode бот в группах получает только ограниченные сообщения, например команды и некоторые специальные случаи.

Создай файл:

Код:
import csv
from pathlib import Path
from datetime import datetime


BASE_DIR = Path(__file__).resolve().parents[2]

PHOTO_LOG_PATH = BASE_DIR / "data" / "raw" / "group_photos.csv"

FIELDNAMES = [
    "datetime",
    "chat_id",
    "chat_title",
    "user_id",
    "username",
    "full_name",
    "message_id",
    "caption",
    "file_id",
    "file_unique_id",
    "saved_path",
]


def clean_text(value: str | None) -> str:
    if not value:
        return ""

    return " ".join(str(value).split())


def save_photo_log(
    chat_id: int,
    chat_title: str,
    user_id: int,
    username: str,
    full_name: str,
    message_id: int,
    caption: str,
    file_id: str,
    file_unique_id: str,
    saved_path: str,
) -> None:
    PHOTO_LOG_PATH.parent.mkdir(parents=True, exist_ok=True)

    file_exists = PHOTO_LOG_PATH.exists()

    row = {
        "datetime": datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
        "chat_id": chat_id,
        "chat_title": clean_text(chat_title),
        "user_id": user_id,
        "username": clean_text(username),
        "full_name": clean_text(full_name),
        "message_id": message_id,
        "caption": clean_text(caption),
        "file_id": file_id,
        "file_unique_id": file_unique_id,
        "saved_path": saved_path,
    }

    with open(PHOTO_LOG_PATH, mode="a", encoding="utf-8-sig", newline="") as file:
        writer = csv.DictWriter(
            file,
            fieldnames=FIELDNAMES,
            quoting=csv.QUOTE_ALL,
        )

        if not file_exists:
            writer.writeheader()

        writer.writerow(row)

Создай:
app/bot/photo_collector.py
Код:
from pathlib import Path
from datetime import datetime

from aiogram.types import Message

from app.config import GROUP_PHOTO_SAVE_DIR
from app.logs.photo_logger import save_photo_log


BASE_DIR = Path(__file__).resolve().parents[2]


def build_user_full_name(message: Message) -> str:
    if not message.from_user:
        return ""

    first_name = message.from_user.first_name or ""
    last_name = message.from_user.last_name or ""

    return f"{first_name} {last_name}".strip()


async def save_group_photo(message: Message) -> str | None:
    """
    Сохраняет фото из Telegram-группы на диск
    и записывает информацию в CSV.

    Важно:
    функция ничего не отвечает пользователю.
    """

    if not message.photo:
        return None

    largest_photo = message.photo[-1]

    chat_id = message.chat.id
    user_id = message.from_user.id if message.from_user else 0

    date_folder = datetime.now().strftime("%Y-%m-%d")

    save_dir = BASE_DIR / GROUP_PHOTO_SAVE_DIR / date_folder
    save_dir.mkdir(parents=True, exist_ok=True)

    username = message.from_user.username if message.
    from_user and message.from_user.username else ""
    full_name = build_user_full_name(message)

    filename = (
        f"{datetime.now().strftime('%H%M%S')}_"
        f"chat_{chat_id}_"
        f"user_{user_id}_"
        f"msg_{message.message_id}_"
        f"{largest_photo.file_unique_id}.jpg"
    )

    saved_path = save_dir / filename

    await message.bot.download(
        largest_photo,
        destination=saved_path,
    )

    save_photo_log(
        chat_id=chat_id,
        chat_title=message.chat.title or "",
        user_id=user_id,
        username=username,
        full_name=full_name,
        message_id=message.message_id,
        caption=message.caption or "",
        file_id=largest_photo.file_id,
        file_unique_id=largest_photo.file_unique_id,
        saved_path=str(saved_path),
    )

    return str(saved_path)

7. Чтобы бот видел фото в группе
В Telegram сделай:
1. Открой @BotFather
2. Напиши /setprivacy
3. Выбери своего бота
4. Выбери Disable
5. Удали бота из группы
6. Добавь бота заново в группу
После отключения privacy mode бот сможет получать обычные сообщения в группе, а не только команды.

9. Что важно по безопасности
Если в группе пользователи отправляют скриншоты, там могут быть:
ИНН
телефон
email
названия компаний
номера документов
ошибки личного кабинета
Поэтому лучше:
не отправлять эти фото третьим API без необходимости
не публиковать датасет
не коммитить data/group_photos в GitHub
Добавь в .gitignore:
data/group_photos/
data/raw/group_photos.csv

10. Проверка
Запусти:
python -m app.main
Отправь фото в группу.
Бот должен:
не ответить ничего
сохранить фото в data/group_photos
добавить строку в data/raw/group_photos.csv
Если фото не сохраняются, почти всегда причина одна: privacy mode не отключён или бот не был заново добавлен в группу после изменения настройки.
