# Что сейчас не так

## 1. `approved.csv` может ломаться из-за старого заголовка

В текущем `review_manager.py` путь идёт в:

```text
data/learning/approved.csv
```

и поля там такие: `case_id`, `datetime`, `language`, `question`, `approved_answer`, `category`, `source_type`. ([GitHub][3])

Но проверка заголовка слабая:

```python
has_header = "case_id" in first_line
```

Если старый CSV уже содержит `case_id`, но остальные столбцы неправильные, код будет продолжать писать в сломанный файл.

---

## 2. `call_center_api.py` сейчас сохраняет в отдельный файл, но не в правильной структуре

Сейчас там указан:

```python
OUTPUT_CSV = "daodesk_cases.csv"
```

и формируются только `ticket_id`, `question`, `answer`, `date`. ([GitHub][4])

Для обучения этого мало. Нужно добавить:

```text
case_id
language
question
approved_answer / admin_answer
category
source_type
source_id
status
```

---

## 3. В `index_builder.py` теряется metadata approved cases

Сейчас `index_builder.py` загружает approved cases, но при добавлении chunk metadata сохраняет только:

```python
{"source": source}
```

из-за этого теряются `source_type`, `category`, `language`. ([GitHub][5])

## Шаг 1. Замени `app/learning/review_manager.py`

Этот файл будет правильно сохранять approved cases и не ломать CSV.

```python
import csv
from pathlib import Path
from datetime import datetime


BASE_DIR = Path(__file__).resolve().parents[2]
APPROVED_CASES_PATH = BASE_DIR / "data" / "learning" / "approved.csv"


APPROVED_FIELDNAMES = [
    "case_id",
    "datetime",
    "language",
    "question",
    "approved_answer",
    "category",
    "source_type",
    "source_id",
    "status",
    "notes",
]


def clean_text(value: str | None) -> str:
    """
    Чистит текст для CSV:
    - убирает лишние переносы строк;
    - убирает лишние пробелы;
    - не ломает структуру CSV.
    """

    if not value:
        return ""

    return " ".join(str(value).split())


def ensure_csv_header(path: Path, fieldnames: list[str]) -> None:
    """
    Проверяет заголовок CSV.
    Если файл существует, но заголовок неправильный,
    делает backup и создаёт новый правильный файл.
    """

    path.parent.mkdir(parents=True, exist_ok=True)

    if not path.exists():
        return

    with open(path, mode="r", encoding="utf-8-sig", newline="") as file:
        reader = csv.reader(file)
        current_header = next(reader, [])

    if current_header != fieldnames:
        backup_path = path.with_name(
            f"{path.stem}_backup_{datetime.now().strftime('%Y%m%d_%H%M%S')}{path.suffix}"
        )
        path.rename(backup_path)
        print(f"Old approved.csv header was wrong. Backup created: {backup_path}")


def save_approved_case(
    case_id: str,
    question: str,
    approved_answer: str,
    language: str = "ru",
    category: str = "general",
    source_type: str = "admin_review",
    source_id: str = "",
    status: str = "approved",
    notes: str = "",
) -> None:
    """
    Сохраняет один проверенный кейс в approved.csv.

    Важно:
    approved.csv должен содержать только хорошие ответы,
    потому что этот файл индексируется в ChromaDB.
    """

    ensure_csv_header(APPROVED_CASES_PATH, APPROVED_FIELDNAMES)

    file_exists = APPROVED_CASES_PATH.exists()

    row = {
        "case_id": clean_text(case_id),
        "datetime": datetime.now().strftime("%Y-%m-%d %H:%M:%S"),
        "language": clean_text(language),
        "question": clean_text(question),
        "approved_answer": clean_text(approved_answer),
        "category": clean_text(category),
        "source_type": clean_text(source_type),
        "source_id": clean_text(source_id),
        "status": clean_text(status),
        "notes": clean_text(notes),
    }

    with open(APPROVED_CASES_PATH, mode="a", encoding="utf-8-sig", newline="") as file:
        writer = csv.DictWriter(
            file,
            fieldnames=APPROVED_FIELDNAMES,
            quoting=csv.QUOTE_ALL,
            extrasaction="ignore",
        )

        if not file_exists:
            writer.writeheader()

        writer.writerow(row)


def approve_case_manually(
    case_id: str,
    question: str,
    approved_answer: str,
    language: str = "ru",
    category: str = "general",
) -> None:
    """
    Старое имя функции оставляем, чтобы test_approve_case.py не сломался.
    """

    save_approved_case(
        case_id=case_id,
        question=question,
        approved_answer=approved_answer,
        language=language,
        category=category,
        source_type="admin_review",
        source_id=case_id,
        status="approved",
    )
```

## Шаг 2. Замени `app/learning/approved_loader.py`

Этот файл будет читать только approved cases со статусом `approved`.

```python
import csv
from pathlib import Path


BASE_DIR = Path(__file__).resolve().parents[2]
APPROVED_CASES_PATH = BASE_DIR / "data" / "learning" / "approved.csv"


def load_approved_cases() -> list[dict]:
    """
    Загружает approved cases из approved.csv
    и превращает их в документы для ChromaDB.
    """

    if not APPROVED_CASES_PATH.exists():
        return []

    documents = []

    with open(APPROVED_CASES_PATH, mode="r", encoding="utf-8-sig", newline="") as file:
        reader = csv.DictReader(file)

        for row in reader:
            status = row.get("status", "approved").strip().lower()

            if status and status != "approved":
                continue

            question = row.get("question", "").strip()
            answer = row.get("approved_answer", "").strip()
            language = row.get("language", "unknown").strip()
            category = row.get("category", "general").strip()
            case_id = row.get("case_id", "").strip()
            source_type = row.get("source_type", "admin_review").strip()
            source_id = row.get("source_id", case_id).strip()

            if not question or not answer:
                continue

            text = f"""
User question:
{question}

Correct support answer:
{answer}
""".strip()

            documents.append(
                {
                    "source": f"{source_type}:{source_id or case_id}",
                    "text": text,
                    "metadata": {
                        "source": f"{source_type}:{source_id or case_id}",
                        "case_id": case_id,
                        "language": language,
                        "category": category,
                        "source_type": source_type,
                        "source_id": source_id,
                    },
                }
            )

    return documents
```

# Шаг 4. Замени `app/faktura_api/call_center_api.py`

Этот файл должен сохранять **сырые кейсы** отдельно:

```text
data/learning/call_center_cases_raw.csv
```

Не в `approved.csv`.

```python
import csv
import time
from pathlib import Path

import requests
from requests.auth import HTTPBasicAuth
from bs4 import BeautifulSoup

from app.config import EMAIL, API_PARAM


BASE_DIR = Path(__file__).resolve().parents[2]

# В .env лучше добавить:
# CALL_CENTER_DOMAIN=https://your-domain.daodesk.com
CALL_CENTER_DOMAIN = "https://d"

CLOSED_STATUS = "closed,resolved"
REQUEST_PAUSE = 0.4

BASE_URL = f"{CALL_CENTER_DOMAIN.rstrip('/')}/api/v2"

OUTPUT_CSV = BASE_DIR / "data" / "learning" / "call_center_cases_raw.csv"

RAW_FIELDNAMES = [
    "case_id",
    "ticket_id",
    "datetime",
    "language",
    "question",
    "admin_answer",
    "category",
    "source_type",
    "source_id",
    "status",
]


auth = HTTPBasicAuth(EMAIL, API_PARAM)


def strip_html(html: str) -> str:
    """
    Убирает HTML-теги из сообщения.
    """

    return BeautifulSoup(html or "", "html.parser").get_text(
        separator=" ",
        strip=True,
    )


def clean_text(value: str | None) -> str:
    """
    Чистит текст для CSV.
    """

    if not value:
        return ""

    return " ".join(str(value).split())


def extract_items(payload) -> list[dict]:
    """
    Универсально достаёт список элементов из ответа API.

    Поддерживает:
    - list
    - {"data": [...]}
    - {"data": {...}}
    - {"items": [...]}
    """

    if isinstance(payload, list):
        return payload

    if not isinstance(payload, dict):
        return []

    data = payload.get("data")

    if isinstance(data, list):
        return data

    if isinstance(data, dict):
        return list(data.values())

    items = payload.get("items")

    if isinstance(items, list):
        return items

    return []


def get_tickets(page: int) -> dict:
    """
    Получает список тикетов.
    """

    params = {"page": page}

    if CLOSED_STATUS:
        params["status_list"] = CLOSED_STATUS

    response = requests.get(
        f"{BASE_URL}/tickets/",
        params=params,
        auth=auth,
        timeout=30,
    )

    response.raise_for_status()

    return response.json()


def get_posts(ticket_id: int) -> list[dict]:
    """
    Получает сообщения внутри тикета.
    """

    response = requests.get(
        f"{BASE_URL}/tickets/{ticket_id}/posts/",
        params={"page": 1},
        auth=auth,
        timeout=30,
    )

    response.raise_for_status()

    return extract_items(response.json())


def build_qa(ticket: dict) -> dict | None:
    """
    Собирает Q/A из одного тикета:
    - вопрос пользователя;
    - ответ сотрудников call center.
    """

    ticket_id = ticket.get("id")

    if not ticket_id:
        return None

    client_id = ticket.get("user_id")

    posts = get_posts(ticket_id)

    if not posts:
        return None

    posts.sort(key=lambda post: post.get("id", 0))

    questions = []
    answers = []

    for post in posts:
        text = strip_html(post.get("text", ""))

        if not text:
            continue

        if post.get("user_id") == client_id:
            questions.append(text)
        else:
            answers.append(text)

    question = clean_text(
        f"{ticket.get('title', '')} {' '.join(questions)}"
    )

    answer = clean_text(
        " ".join(answers)
    )

    if not question or not answer:
        return None

    return {
        "case_id": f"callcenter_{ticket_id}",
        "ticket_id": ticket_id,
        "datetime": ticket.get("date_created", ""),
        "language": "ru",
        "question": question,
        "admin_answer": answer,
        "category": "call_center",
        "source_type": "call_center",
        "source_id": str(ticket_id),
        "status": "raw",
    }


def save_call_center_cases(max_pages: int | None = None) -> None:
    """
    Сохраняет реальные кейсы call center в отдельный raw CSV.
    """

    OUTPUT_CSV.parent.mkdir(parents=True, exist_ok=True)

    rows = []
    page = 1

    while True:
        data = get_tickets(page)
        tickets = extract_items(data)

        if not tickets:
            break

        for ticket in tickets:
            qa = build_qa(ticket)

            if qa:
                rows.append(qa)
                print(
                    f"Ticket {qa['ticket_id']}: {qa['question'][:60]}..."
                )

            time.sleep(REQUEST_PAUSE)

        pagination = data.get("pagination", {}) if isinstance(data, dict) else {}
        total_pages = pagination.get("total_pages", page)

        print(f"Page {page}/{total_pages} processed")

        if max_pages and page >= max_pages:
            break

        if page >= total_pages:
            break

        page += 1
        time.sleep(REQUEST_PAUSE)

    with open(OUTPUT_CSV, mode="w", encoding="utf-8-sig", newline="") as file:
        writer = csv.DictWriter(
            file,
            fieldnames=RAW_FIELDNAMES,
            quoting=csv.QUOTE_ALL,
        )

        writer.writeheader()
        writer.writerows(rows)

    print(f"\nReady: {len(rows)} cases saved in {OUTPUT_CSV}")


if __name__ == "__main__":
    save_call_center_cases()
```

# Шаг 5. Добавь импорт raw cases в approved.csv

Создай новый файл:

```text
app/learning/import_call_center_to_approved.py
```

Код:

```python
import csv
from pathlib import Path

from app.learning.review_manager import save_approved_case


BASE_DIR = Path(__file__).resolve().parents[2]

RAW_CALL_CENTER_PATH = BASE_DIR / "data" / "learning" / "call_center_cases_raw.csv"


def import_call_center_to_approved(
    min_question_len: int = 10,
    min_answer_len: int = 30,
    auto_approve: bool = False,
) -> None:
    """
    Импортирует call center cases в approved.csv.

    Важно:
    auto_approve=False по умолчанию.
    Лучше сначала посмотреть raw cases вручную.
    """

    if not RAW_CALL_CENTER_PATH.exists():
        print(f"File not found: {RAW_CALL_CENTER_PATH}")
        return

    imported = 0
    skipped = 0

    with open(RAW_CALL_CENTER_PATH, mode="r", encoding="utf-8-sig", newline="") as file:
        reader = csv.DictReader(file)

        for row in reader:
            question = row.get("question", "").strip()
            answer = row.get("admin_answer", "").strip()
            ticket_id = row.get("ticket_id", "").strip()
            case_id = row.get("case_id", f"callcenter_{ticket_id}").strip()

            if len(question) < min_question_len or len(answer) < min_answer_len:
                skipped += 1
                continue

            if not auto_approve:
                skipped += 1
                continue

            save_approved_case(
                case_id=case_id,
                question=question,
                approved_answer=answer,
                language=row.get("language", "ru"),
                category=row.get("category", "call_center"),
                source_type="call_center",
                source_id=ticket_id,
                status="approved",
                notes="Imported from call center API.",
            )

            imported += 1

    print(f"Imported: {imported}")
    print(f"Skipped: {skipped}")


if __name__ == "__main__":
    # В первый раз лучше не автоаппрувить.
    # После проверки raw CSV можно поставить auto_approve=True.
    import_call_center_to_approved(auto_approve=False)
```

# Нужно ли сохранять call center ответы сразу в approved.csv?

**Нет, не сразу.**

Лучший вариант:

```text
call_center_cases_raw.csv  — всё как пришло из API
approved.csv               — только проверенные хорошие ответы
```

Почему:

```text
1. Ответы операторов могут быть короткие.
2. Там могут быть персональные данные.
3. Может быть мусор из HTML.
4. Может быть неполный ответ.
5. Если сразу отправить это в Chroma, бот начнёт учиться на плохих данных.
```

Да, это правильный следующий шаг. Сейчас админ-панель лучше перестроить так, чтобы она не была одним огромным `admin_handler.py`, а стала отдельным модулем с маленькими файлами по функциям.

Главная идея:

```text
admin panel
    ↓
создать постоянный Q/A
    ↓
создать временную проблему / инцидент
    ↓
активировать режим “отвечать так на все похожие вопросы”
    ↓
бот сначала проверяет active_incidents
    ↓
если совпало — отвечает готовым ответом
    ↓
если нет — идёт в Faktura API / Chroma / Qwen
```

# 3. Постоянные Q/A и временные проблемы нужно разделить

Это очень важно.

У тебя будут два разных типа знаний.

---

## Тип 1. Постоянные знания

Например:

```text
Вопрос:
Как создать ТТН?

Ответ:
Для создания ТТН перейдите...
```

Это можно сохранять в:

```text
data/learning/approved.csv
```

Потому что это стабильная база знаний.

---

## Тип 2. Временная проблема / инцидент

Например:

```text
Проблема:
Сбой в soliq.uz

Ответ:
Сейчас наблюдаются временные сбои на стороне soliq.uz. 
Рекомендуем повторить отправку позже.
```

Это **нельзя** сохранять только в `approved.csv`, потому что через день проблема закончится, а бот продолжит так отвечать.

Для этого нужен отдельный файл:

```text
data/admin/active_incidents.csv
```

или лучше:

```text
data/admin/incidents.csv
```

# 7. Функция “отвечать так на все вопросы по этой проблеме”

Для этого нужен отдельный файл:

```text
app/core/emergency_answer_checker.py
```

Он будет работать **до Chroma, до API и до Qwen**.

Схема:

```text
Вопрос пользователя
    ↓
Проверить active incidents
    ↓
Если совпали keywords
    ↓
Сразу вернуть готовый ответ
    ↓
Не вызывать Qwen / Gemini
```

То есть в `bot_engine.py` порядок должен быть такой:

```python
def process_user_question(question: str, ...):
    language = detect_language(question)

    emergency_answer = check_active_incidents(question, language)

    if emergency_answer:
        return {
            "answer": emergency_answer,
            "sources": [
                {
                    "source": "Admin active incident",
                    "source_type": "incident",
                }
            ],
            "sent_to_review": False,
        }

    # дальше уже обычный путь:
    # 1. Faktura API
    # 2. ChromaDB
    # 3. Qwen/Gemini
```

Это решит твой кейс:

```text
Админ:
“Сейчас сбой в Soliq. Всем, кто спрашивает про отправку/ошибку/soliq, отвечать вот так...”

Пользователь:
“Почему не отправляется счет-фактура?”

Бот:
“Сейчас наблюдаются временные сбои на стороне soliq.uz...”
```

# 15. Services отдельно от handlers

Handlers должны только принимать сообщения.

Плохой вариант:

```python

async def add_incident(message):
    open csv
    write row
    validate
    format
    save
```

Хороший вариант:

```python
@router.message(...)
async def add_incident(message):
    incident_service.save_incident(...)
```

# 16. Как это будет работать в bot_engine

Итоговая логика ответа бота должна быть такой:

```text
1. Пользователь задал вопрос
2. Проверяем активные проблемы
3. Если проблема найдена — отвечаем готовым admin answer
4. Если нет — проверяем Faktura API
5. Если API не нужен — ищем в ChromaDB
6. Генерируем ответ через Qwen/Gemini
7. Логируем вопрос и ответ
8. Если ответ слабый — отправляем в review
```

# 17. Финальная архитектура знаний

В итоге у тебя будет 4 уровня знаний:

```text
1. active incidents
   Срочные временные ответы.
   Проверяются первыми.

2. Faktura API
   Реальные данные из API.

3. approved.csv
   Проверенные ответы операторов и админов.

4. PDF база
   Официальные инструкции Faktura Wiki.
```

Порядок приоритета:

```text
active incidents
    ↓
Faktura API
    ↓
approved cases
    ↓
PDF knowledge base
    ↓
Qwen/Gemini
```
